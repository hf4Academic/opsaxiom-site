怎么贡献你验证过的 Skill

两条通道殊途同归：都落到本仓库的一个 Pull Request，CI 自动质检，维护者合入即上架。

先装客户端（3 步，约 5 分钟）

git clone https://github.com/hf4Academic/opsaxiom.git && cd opsaxiom
./install.sh                              # 装依赖、把 opsaxiom 命令软链进 PATH、自检
export PATH="$HOME/.local/bin:$PATH"      # 若安装末尾提示 PATH，加这行（可写进 ~/.bashrc）

装完敲 opsaxiom doctor：🟢 全绿即可用（🟡 只是可选连接器没装，不影响）。 气隙/内网环境用离线包：./install.sh --offline。想在对话里排查，另接一个模型即可 （opsaxiom model，或 pi 界面里 /login）。客户端源码见 https://github.com/hf4Academic/opsaxiom。

通道一：客户端（推荐，全程引导）

opsaxiom skill from-session <会话id>   # 把你刚做完的排查自动变成草稿
opsaxiom skill lint <草稿>             # 校验 + 缺口清单，补完
# 仿真验证晋级到 🔵 后：
opsaxiom hub push <skill-id>           # 打包 bundle（pi 界面里 /publish 上下键选）

得到 bundle 后：fork 本仓库 → 解包到 skills/<id>/<version>/ → 提 PR。

通道二：纯网页（不装客户端）

1. Fork 本仓库；

2. 在你的 fork 里进入 skills/，用网页 Add file → Upload files 上传你的 <skill-id>/<version>/ 目录（skill.yaml + tests/ + attestations/）；

3. 发起 Pull Request——PR 就是发布表单：描述你在什么环境验证过、结果如何。

收录标准（CI 自动检查 + 维护者评审）

① 校验全绿（结构 + 语义 S1–S13 + 命令语法树）；② 成熟度 ≥ 🔵 sim_verified（⚪draft 拒收）； ③ 实地验证记录（attestation）签名有效；④ 任何写操作必须带经过验证的回滚方案。

完整政策见 policy.md。
