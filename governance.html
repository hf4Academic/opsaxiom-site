治理与投诉

发现某个 Skill 误导、翻车、过时或有安全隐患？投诉它，进入治理流程。

怎么投诉（👎）

每个 Skill 详情页都有 👎 投诉此 Skill 按钮——它会带着 Skill id 打开一个 GitHub Issue（标签 report），你只需写清：什么问题 + 什么环境 + 怎么复现。 不需要账号以外的任何东西。

投诉之后发生什么（治理流程）

1. 核实：维护者按 Issue 复现（必要时在仿真环境重跑该 Skill 的 tests）；

2. 处置（按严重度）：修正（提 PR 修复）／降级（撤销徽章，如 🟢→🔵）／ 下架（git revert 移出 registry——留痕不删档，历史可查）；

3. 公示：处置结论回帖在原 Issue，负面 attestation 同样入库（负面记录是社区的宝贵信号）。

谁在治理

维护者 = 本仓库有合入权限的人；可信签名者名单在 keyring/trusted.pub， 加入需维护者双人复核（PR 形式）。所有治理动作都是 git 提交——可追溯、可回滚。
