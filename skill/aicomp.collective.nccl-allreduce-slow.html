决策树

入口：measure_bw

[check] measure_bw — 用 nccl-tests 量测 busbw 与期望差距
· 当 busbw_gbps < expected_busbw * 0.5 → severe_exit
· 当 busbw_gbps < expected_busbw * 0.8 → check_transport
· 当 busbw_gbps >= expected_busbw * 0.8 → ok_exit
· 否则 → escalate
⚠ 先用 nccl-tests 客观量测 busbw，别凭训练慢就猜 NCCL——训练慢也可能是数据加载/CPU 瓶颈。busbw 对比理论峰值才能定位是不是通信问题
⚠ NCCL_DEBUG=INFO 能看它实际选的传输(NVLink/IB/PCIe/SHM)和拓扑——用错传输(比如该走 IB 却走了 TCP)是慢的常见根因
[done] ok_exit — busbw 达期望 80% 以上，通信带宽正常。训练若仍慢，瓶颈可能在数据加载/计算/CPU，转 aicomp/training/gpu-util-low 或查数据管道。
[check] check_transport — 中度偏慢——检查 NCCL 选用的传输与拓扑
· 当 using_tcp > 0 → tcp_fallback_exit
· 当 using_ib > 0 → ib_tune_exit
· 否则 → escalate
⚠ 看到 NET/Socket(TCP) 而非 NET/IB 说明 NCCL 没走 IB(RDMA)，退化到 TCP——带宽暴跌。多为 NCCL_IB_HCA/网络配置问题
[done] tcp_fallback_exit — NCCL 退化到 TCP(未走 IB/RDMA)，带宽大幅下降。方向：检查 NCCL_IB_HCA/NCCL_SOCKET_IFNAME 配置、IB 设备是否可见(ibstat)、GPUDirect RDMA 是否启用、容器是否透传了 IB
[done] ib_tune_exit — 已走 IB 但带宽仍不足。方向：检查 IB 链路是否降速(转 aicomp/fabric/ib-link-down)、NCCL 拓扑/通道数、是否开启 GPUDirect(NCCL_NET_GDR_LEVEL)、跨 NUMA/PCIe 交换
[done] severe_exit — busbw 不足期望一半，严重异常。优先查：是否根本没走 NVLink/IB(退化 PCIe/TCP)、是否有卡降频或 IB 链路故障、是否有慢节点拖累集合通信(木桶效应，转 aicomp/training/slow-node)。
[escalate] escalate — 打包升级：nccl-tests busbw、NCCL_DEBUG=INFO 传输选择、IB/NVLink 状态、拓扑、期望峰值。
