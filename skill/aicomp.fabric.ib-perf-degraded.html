决策树

入口：check_rate

[check] check_rate — 查 IB 端口速率是否降级
· 当 output.value > 0 → check_errors
· 当 output.value == 0 → check_errors
· 否则 → check_errors
⚠ IB 端口协商到低速率(如 HDR 卡跑成 EDR/FDR)=线缆/端口问题，带宽直接腰斩；先确认速率对不对再查别的
[check] check_errors — 查 IB 端口错误计数
· 当 output.value > 0 → done_errors
· 当 output.value == 0 → done_config
· 否则 → escalate
⚠ SymbolError/PortRcvErrors 增长=物理链路质量问题(线缆/光模块)；RDMA 慢也可能是 GDR(GPUDirect RDMA)没开或 NUMA 亲和不对，非硬件
[done] done_errors — IB 有错误计数或速率降级。查线缆/光模块/端口，换线重插；速率不对核对两端与交换机的速率配置。
[done] done_config — IB 端口正常无错误，RDMA 慢多为配置问题。检查 GDR 是否启用、GPU 与 IB 卡的 NUMA 亲和(NCCL_IB_HCA/绑核)、MTU 是否一致。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
