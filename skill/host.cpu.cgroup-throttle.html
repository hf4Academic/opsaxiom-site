决策树

入口：check_throttled

[check] check_throttled — 查 cgroup CFS 被限流次数
· 当 output.throttled > 0 → check_periods
· 当 output.throttled <= 0 → done_no_throttle
· 否则 → check_periods
⚠ nr_throttled 增长=进程在 CFS 周期内用完配额被强制暂停到下周期,表现为周期性卡顿/毛刺,哪怕整机 CPU 还很闲——这是'限额'不是'没资源',别去查整机负载
[check] check_periods — 看被限流的时间占比是否高
· 当 output.tusec > 1000000 → done_heavy_throttle
· 当 output.tusec <= 1000000 → done_light_throttle
· 否则 → escalate
⚠ 限流时间占比高说明 limit 设得对业务太紧;调大 cpu.max/limits.cpu 或放宽配额,多线程应用还要注意配额和线程数匹配(配额小但线程多,极易频繁触顶)
[done] done_no_throttle — 无 CFS 限流。容器慢另查(整机负载/内存/IO/应用自身/其它 cgroup 控制器)。
[done] done_heavy_throttle — 严重 CPU 限流,配额对业务太紧。调大容器 CPU limit/cpu.max,或减少并发线程数与配额匹配;整机有余量时适度放宽,消除周期性毛刺。
[done] done_light_throttle — 轻度限流,偶发触顶。观察是否业务峰值短暂超配额;若毛刺可接受可不动,否则小幅上调配额留出余量,避免峰值被 CFS 掐。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
