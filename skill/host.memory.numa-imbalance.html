决策树

入口：check_foreign

[check] check_foreign — 查跨 NUMA 节点访问计数
· 当 output.foreign > 1000000 → check_free
· 当 output.foreign <= 1000000 → done_numa_ok
· 否则 → check_free
⚠ numa_foreign 高=进程在别的节点分了内存(本地节点不够),跨节点访问延迟高一截;大内存进程(DB/JVM)没做 NUMA 亲和时最容易踩
[check] check_free — 看各节点空闲内存是否悬殊
· 当 output.value > 1 → done_imbalanced
· 当 output.value <= 1 → done_single_node
· 否则 → escalate
⚠ 单节点内存被某进程吃满、另一节点空闲,会触发跨节点分配甚至该节点内的回收抖动;绑 NUMA(numactl --membind/--cpunodebind)或交给自动均衡要按业务权衡
[done] done_numa_ok — 跨节点访问不高。内存慢另查(带宽/大页/交换/具体进程访存模式)。
[done] done_imbalanced — NUMA 节点间不均衡致跨节点访问。给大内存进程做 NUMA 亲和(numactl 绑 CPU+内存到同节点)、或按节点均分负载;数据库/JVM 尤其受益于亲和绑定。
[done] done_single_node — 疑似单节点或数据不足。确认机器是否真多 NUMA 节点;若是,核对进程内存策略(interleave vs bind),避免关键进程内存溢到远端节点。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
