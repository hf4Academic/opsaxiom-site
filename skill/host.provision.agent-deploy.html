决策树

入口：idempotency_check

[check] idempotency_check — 幂等检查——是否已部署
· 当 any(lines[] matches 'active') → already_deployed
· 当 any(lines[] matches 'PORT_USED') → port_conflict_exit
· 当 any(lines[] matches 'absent') → install_probe
· 否则 → install_probe
⚠ 批量部署第一原则是幂等：已装的跳过、端口占用的报错、干净的才装。不做幂等检查的批量下发会在'部分已装'的环境里翻车
⚠ 9100 被别的服务占用时不要强装——先查清占用者，避免撞端口引发更大故障
[done] already_deployed — node_exporter 已在运行，跳过(幂等)。若需升级版本，走单独的升级流程而非重装。
[escalate] port_conflict_exit — 9100 端口已被其他进程占用但非 node_exporter。需人工确认占用者身份后再决定换端口或清理，不自动处置。
[action] install_probe — 安装并启动 node_exporter
⚠ 部署工具须内置校验：二进制 checksum 校验、架构匹配检查。来源不可信的二进制不下发(供应链风险)
⚠ 回滚=完整卸载(停服务+删单元+删二进制)，确保回滚后主机回到部署前状态，不留残留
[check] verify_scrape — 确认监控系统能抓取到
· 当 output.value > 0 → done_ok
· 否则 → escalate
⚠ 本机 /metrics 通不代表 Prometheus 能抓到——还要确认防火墙放行、Prometheus 已加该 target。本 Skill 只保证本机侧就绪
[done] done_ok — node_exporter 部署完成且本机 /metrics 正常。后续：在 Prometheus 侧加入该 target 并确认防火墙放行 9100(建议仅对监控网段开放)。
[escalate] escalate — 打包升级：部署日志、systemd 状态、/metrics 探测结果、端口占用情况。
