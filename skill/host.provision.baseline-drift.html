决策树

入口：check_sysctl

[check] check_sysctl — 抽查关键内核参数是否偏离基线
· 当 output.swappiness > 30 → done_drift_found
· 当 output.swappiness <= 30 → check_selinux
· 否则 → check_selinux
⚠ 基线值因业务而异（此处 swappiness 阈值仅示例）——真实基线应来自你的标准，本 Skill 演示漂移检测方法而非规定具体值
[check] check_selinux — 查安全模块状态是否被关
· 当 output.value > 0 → done_security_off
· 当 output.value == 0 → done_baseline_ok
· 否则 → escalate
⚠ SELinux 被临时 setenforce 0 不会持久但会留隐患；被改 config 则重启后仍关——两者危害不同，要区分临时态与配置态
[done] done_drift_found — 检测到内核参数偏离基线。对照标准基线恢复（sysctl -w 临时、写 sysctl.conf 持久）；查审计日志追谁改的、为什么。
[done] done_security_off — 安全模块被关(disabled/permissive)。确认是否有意为之；无正当理由则恢复 enforcing，并排查是谁在什么时候关的。
[done] done_baseline_ok — 抽查项符合基线。全量核查建议用配置管理工具(ansible/salt)跑一次 diff，本 Skill 只做快速抽检。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
