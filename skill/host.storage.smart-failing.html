决策树

入口：scan_health

[check] scan_health — 逐盘 SMART 总体健康判定
· 当 output.failed_count >= 1 → check_attrs
· 当 output.failed_count == 0 → check_attrs
· 否则 → check_attrs
⚠ SMART PASSED 不代表盘健康！总体判定只反映厂商预设阈值，关键属性（pending/uncorrectable）越界前常仍显示 PASSED——必须看属性明细，这是本 Skill 存在的核心原因
⚠ RAID 卡后面的物理盘直连扫不到，要 `smartctl -A -d megaraid,N /dev/sda`（N 为盘位）
[check] check_attrs — 读取关键属性区分根因
· 当 output.pending > 0 or output.uncorrectable > 0 → done_dying
· 当 output.reallocated > 50 → done_plan_replace
· 当 output.crc_errors > 100 → done_check_cable
· 当 output.pending == 0 and output.uncorrectable == 0 and output.reallocated <= 50 and output.crc_errors <= 100 → done_healthy
· 否则 → escalate
⚠ NVMe 属性字段名不同（Percentage_Used/Media_and_Data_Integrity_Errors），解析器两种都认
⚠ Current_Pending_Sector>0 是最危险信号：有扇区读不出来正等待重映射，盘随时可能整片失效
[done] done_dying — 盘即将失效：有待重映射/不可修复扇区（pending={{output.pending}}、uncorrectable={{output.uncorrectable}}）。 **立即人工处置（Agent 不代执行）：① 先 rsync/备份
[done] done_plan_replace — 重映射扇区偏多（reallocated={{output.reallocated}}>50）。盘在缓慢劣化，尚可用但寿命有限——排入计划性换盘窗口，换前确保有备份。
[done] done_check_cable — CRC 错误多（crc_errors={{output.crc_errors}}>100）。**这是 SATA 线缆/接口/背板问题，不是盘坏！** 换盘白花钱。处置：重插或更换数据线、检查背板供电与接口氧化。CRC 错误只影响传输、
[done] done_healthy — 关键属性均在安全范围（无 pending/uncorrectable、重映射少、CRC 低）。当前无换盘必要，建议纳入定期 SMART 巡检基线跟踪趋势。
[escalate] escalate — 打包升级：各盘 -H 总判 + -A 全属性、盘型号与接口类型、dmesg 近期 I/O error。
