决策树

入口：check_usage

[check] check_usage — 查看 journal 占用
· 当 output.take > 1000000000 → check_config
· 当 output.take <= 1000000000 → done_small
· 否则 → check_config
⚠ journalctl --disk-usage 的输出单位是字节，别把 1.0G 当成 1
[check] check_config — 查看保留上限配置
· 当 output.count == 0 → done_no_limit
· 当 output.count > 0 → done_has_limit
· 否则 → escalate
⚠ 没配 SystemMaxUse 时 journald 默认最多用 10% 分区，大盘上就是几十 GB
[done] done_small — journal 占用不大（<1GB），空间问题另有元凶，转查 disk-full。
[done] done_no_limit — 未配置 journal 上限，占用会持续增长。**立即缓解（人工执行，不可逆）：journalctl --vacuum-size=500M 清理旧日志；根治：在 journald.conf 配 SystemMaxUse=500M 后 sys
[done] done_has_limit — 已配上限但仍占满，可能是配置未生效或值过大。核对 SystemMaxUse 实际值并重启 journald 使其生效。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
