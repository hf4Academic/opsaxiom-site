决策树

入口：check_ready

[check] check_ready — 查堆积最多的队列
· 当 output.value > 10000 → check_consumers
· 当 output.value <= 10000 → done_backlog_ok
· 否则 → check_consumers
⚠ 消息堆积会占内存，超过 vm_memory_high_watermark(默认 40%)会阻塞生产者(流控)——堆积不处理最终会连生产一起卡
[check] check_consumers — 看该队列有无消费者
· 当 output.value == 0 → done_no_consumer
· 当 output.value > 0 → done_slow_consumer
· 否则 → escalate
⚠ 消费者数为 0=没人消费(消费者挂了/没连上)，堆积必然；有消费者仍堆=消费处理慢或prefetch 太小限制了吞吐
[done] done_backlog_ok — 队列堆积不严重。若有告警看是否某队列个别堆积或内存其它占用。
[done] done_no_consumer — 堆积队列没有消费者。查消费者进程是否挂了/连接断了/绑定错队列；恢复消费者后积压会被消化。
[done] done_slow_consumer — 有消费者但消费慢。优化消费逻辑、增加消费者、调大 prefetch_count 提升吞吐(权衡公平性)；确认没有 unacked 消息卡住。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
