决策树

入口：check_slowlog

[check] check_slowlog — 查慢日志条数
· 当 output.value > 0 → check_bigkey
· 当 output.value == 0 → done_no_slow
· 否则 → check_bigkey
⚠ slowlog 只记超过 slowlog-log-slower-than(默认 10ms)的命令；有条目说明确有慢命令，但阈值配太大可能漏记
[check] check_bigkey — 看是否存在大 key(慢命令常因)
· 当 output.value > 0 → done_bigkey
· 当 output.value == 0 → done_slow_cmd
· 否则 → escalate
⚠ O(N) 命令(KEYS/HGETALL/SMEMBERS 大集合)是慢命令首因——尤其 KEYS 会阻塞整个实例。禁用 KEYS、用 SCAN 替代
[done] done_no_slow — 无慢日志。延迟高另查(网络/fork/内存换页)。
[done] done_bigkey — 存在大 key，是慢命令的根源。拆分大 key(大 hash/set 分片)、避免对大 key 用 O(N) 命令；删除大 key 用 UNLINK(异步)而非 DEL。
[done] done_slow_cmd — 有慢命令但非大 key 主导。slowlog get 看具体命令，禁用 KEYS 等 O(N) 命令、优化 Lua 脚本、避免大范围 ZRANGE。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
