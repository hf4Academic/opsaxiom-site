决策树

入口：read_state

[check] read_state — 邻接卡在哪个状态
· 当 state == 'Init' → init_exit
· 当 state == 'ExStart' or state == 'Exchange' → mtu_exit
· 当 state == 'Down' → down_exit
· 当 state == 'Full' or state == '2Way' → normal_exit
· 否则 → escalate
⚠ OSPF 状态机卡点直接指向根因：Init=单向收到 hello(对端没回，查对端/单通)；ExStart/Exchange 卡住=经典 MTU 不匹配；Down=完全没 hello
⚠ 2Way 在多路访问网络(以太网)对 DR/BDR 之外的邻居是正常终态，别当故障
[done] down_exit — Down：收不到对端 hello。查接口是否 up、OSPF 是否在该接口启用、hello/dead 定时器与区域号是否两端一致、认证是否匹配。
[done] init_exit — Init：本端收到对端 hello 但对端没认到本端(单向)。多为单通(查双向可达)、或对端的 hello 参数/认证/子网掩码不匹配把本端 hello 丢了。
[done] mtu_exit — 卡在 ExStart/Exchange：最经典的 OSPF 故障——两端接口 IP MTU 不匹配导致 DBD 交换失败。核对两端接口 MTU 是否一致(转 mtu 相关)，或配置 ospf mtu-ignore 临时绕过(治标)。
[done] normal_exit — 邻接已 Full/2Way(正常)。若路由仍有问题，转 network/routing 查 LSA/路由计算，而非邻接。
[escalate] escalate — 打包升级：邻接状态、两端接口 MTU/区域/定时器/认证、接口状态。
