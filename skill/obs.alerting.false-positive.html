决策树

入口：check_real

[check] check_real — 判断告警是否反映真实越界
· 当 flapping_count > 5 → flapping_exit
· 当 firing > 0 and metric_value < threshold → threshold_exit
· 当 firing > 0 → real_exit
· 否则 → escalate
⚠ 别把'真问题'当误报关掉——先看当前指标值 vs 阈值：值确实没越界才叫误报，越界了就是真告警
⚠ 抖动(flapping)不是误报而是配置问题：指标在阈值附近反复穿越→加 for 持续时间或滞回(hysteresis)，而非删告警
⚠ 误报的典型来源：静态阈值不适应业务波动、聚合窗口太短、单点采样毛刺
[done] threshold_exit — 当前指标值未越界却在 firing，属阈值/规则不当误报。方向：改用更贴合的阈值(如分时段/基线偏离)、拉长评估窗口、或换成比率类而非绝对值类判据。调整前确认历史上它是否漏过真事件。
[done] flapping_exit — 告警在阈值附近反复抖动({{output.flapping_count}} 次)，是抖动而非误报。方向：加 `for:` 持续时间(如 5m)要求持续越界才触发、引入滞回(触发阈≠恢复阈)、或对指标做适度平滑。不要简单删告警。
[done] real_exit — 指标确实越界，这不是误报而是真问题。方向：按该告警对应的处置 Skill 排查根因，别急着调阈值把真信号屏蔽掉。
[escalate] escalate — 打包升级：告警规则表达式、当前指标值、阈值、for 配置、近段触发/恢复历史，供人工判定误报性质。
