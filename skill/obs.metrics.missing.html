决策树

入口：classify_missing

[check] classify_missing — 分类指标缺失形态
· 当 target_up == 0 → target_down_exit
· 当 series_count == 0 → never_exposed_exit
· 当 last_scrape_age_s > 300 → stale_exit
· 当 series_count > 0 and last_scrape_age_s <= 300 → present_exit
· 否则 → escalate
⚠ 分清三种'没数据'：target 挂了(抓取端点不通)、指标从没被暴露(名字错/exporter 没这个指标)、曾有数据但采集停了(scrape 失败/stale)——处置完全不同
⚠ 指标名大小写/label 写错会让人误判为'缺失'，先用 promtool 按前缀模糊查确认真实指标名
[done] target_down_exit — 指标缺失是因为它所在的 target 处于 down。转 obs/metrics/target-down 排查 exporter 为何抓不到；target 恢复后指标自然回来。
[done] never_exposed_exit — 该指标名在库里没有任何 series，多为指标名/label 写错，或 exporter 根本没暴露这个指标(版本差异/未开启对应 collector)。方向：确认真实指标名(promtool 前缀查)、检查 exporter 是否需要开启
[done] stale_exit — 指标曾有数据但已 {{output.last_scrape_age_s}}s 未更新(stale)，说明采集近期停了。方向：查该 target 的 scrape 是否开始失败(转 target-down)、relabel/drop 规则是否
[done] present_exit — 指标实际有数据且新鲜，'缺失'可能是查询语句/看板 label 选择器问题，或看的时间范围内恰好无采样。核对 PromQL 与看板变量。
[escalate] escalate — 打包升级：指标名、series 数、最近 scrape 时间、对应 target 状态，供人工判定缺失环节。
