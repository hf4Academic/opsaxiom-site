决策树

入口：check_rootlogin

[check] check_rootlogin — 查是否允许 root 直接登录
· 当 output.value > 0 → done_root_allowed
· 当 output.value == 0 → check_passwd
· 否则 → check_passwd
⚠ 用 sshd -T 看'生效值'而非 grep 配置文件——Match 块/Include 会覆盖，直接读文件常误判;PermitRootLogin yes 让 root 成暴力破解首要目标,基线应为 no 或 prohibit-password
[check] check_passwd — 查是否允许密码登录
· 当 output.value > 0 → done_passwd_on
· 当 output.value == 0 → done_baseline_ok
· 否则 → escalate
⚠ 开启密码登录=可被暴力破解/撞库;基线应仅用密钥(PasswordAuthentication no)。关之前务必先确认已配好可用密钥,否则会把自己锁在外面
[done] done_root_allowed — 允许 root 直接登录(高风险)。改 PermitRootLogin 为 no/prohibit-password,改用普通账号+sudo;改前确认有可用的非 root 登录方式再重载 sshd。
[done] done_passwd_on — root 已受限但仍允许密码登录。评估改为仅密钥认证(PasswordAuthentication no);先给相关账号配好密钥并验证可登录,再关闭密码认证,防止锁死。
[done] done_baseline_ok — SSH 基线较好(禁 root 直登+禁密码)。可进一步核对算法套件、登录源限制(AllowUsers/防火墙)、失败重试与 MaxAuthTries。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
