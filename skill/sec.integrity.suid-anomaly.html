决策树

入口：check_suid_count

[check] check_suid_count — 统计系统内 SUID 文件数
· 当 output.value > 40 → check_writable_dirs
· 当 output.value <= 40 → check_writable_dirs
· 否则 → check_writable_dirs
⚠ SUID 文件以属主权限运行,是提权的常见跳板;应与已知基线清单比对,重点看'不该有 SUID'的程序(如 shell 解释器、find、nmap、cp)——它们带 SUID 往往是被人为植入的提权后门
[check] check_writable_dirs — 查可疑位置(可写目录)下的 SUID 文件
· 当 output.value > 0 → done_suspicious_suid
· 当 output.value == 0 → done_suid_baseline
· 否则 → escalate
⚠ /tmp、/home、/dev/shm 这类用户可写目录里出现 SUID 文件几乎必然是恶意的——正常系统的 SUID 都在系统目录且属主 root,可写目录里的属高危,优先处置
[done] done_suspicious_suid — 在用户可写目录发现 SUID 文件(高危,疑似提权后门)。隔离取证该文件(勿直接删,先留证),核实哈希/来历,清除并排查入侵链(是谁放的、怎么进来的)。
[done] done_suid_baseline — 可写目录无异常 SUID。将系统目录 SUID 清单与基线比对,重点核对非标准程序;建立 SUID 基线并定期 diff,新增项及时审查。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
