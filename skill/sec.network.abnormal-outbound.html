决策树

入口：check_estab

[check] check_estab — 统计对外已建立连接数
· 当 output.value > 0 → check_proc
· 当 output.value == 0 → done_no_outbound
· 否则 → check_proc
⚠ 排查外连先看'谁在连、连去哪'两件事;正常业务外连能对应到已知服务(DB/API/仓库),对不上进程或目的地陌生的才可疑,别把正常业务连接当威胁
[check] check_proc — 查是否有无属主/可疑进程发起外连
· 当 output.value > 0 → done_suspicious_conn
· 当 output.value == 0 → done_outbound_known
· 否则 → escalate
⚠ shell/脚本解释器/nc 直接持有对外长连接是反连(reverse shell)的典型特征;而 curl/wget 常连=可能是定时下载器(挖矿/木马拉马)。看它连的目的地和父进程链
[done] done_no_outbound — 无对外已建立连接。若怀疑外连威胁,可再抓一段时间(周期性外连可能间歇);或结合防火墙出站日志看有无被拦的尝试。
[done] done_suspicious_conn — shell/脚本/nc 类进程持有外连(疑似反连或恶意下载)。取证(记录目的IP/端口/进程树/打开文件),隔离该主机,排查入侵入口与持久化(计划任务/启动项)。
[done] done_outbound_known — 外连均由常规服务发起。逐一核对目的地与业务预期一致,用出站防火墙白名单收窄(只放行必要目的地),降低被反连/外传的面。
[escalate] escalate — 打包升级：已采集的事实与走过的路径。
